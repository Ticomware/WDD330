Week 9: The Window Object and HTML5

Notes

Chapter 9: The Window Object

  • The window object in JS gives information about the browser. The methods and properties about the browser are also said to be part of the Browser Object Model.
  • In JS, when working outside of the browser, there likely is a global object that works similarly to window.
  • Properties and methods that are built into JS are part of the global object, as are any globally defined properties or methods.
  • By using the global object, you can avoid reference errors if a property is not defined.
  • alert(), confirm(), and prompt() are used to show a messagebox.
  • UA strings are now deprecated, but will likely continue to be supported for forever.
  • window.location contains info about the current page, including href
  • window.history contains info about pages backwards and forwards in the browser.
  • window.history.go() can be used to programmatically navigate the user forwards, backwards, or refresh the page.
  • window.open() is used to open a tool window as part of a webapp.
  • window.screen will give the resolution of the display or the size of the browser window.
  • window.document is the JS DOM object.
  • Using window.document.cookie, JS can append to, update, or read cookies.
  • Cookie can be assigned an expiry date, by default lasts until tab is closed. Can set domain of a cookie also to access cookies in subdomains.
  • Cookies can be set to only transmit over HTTPS by using the secure attribute. Cookies can be deleted by setting the expiry date in the past.
  • window.setTimeout calls a function after a timeout, can be cancelled with clearTimeout()
  • window.setInterval calls a function repeatedly every time interval specified.

Chapter 14: HTML5 APIs

  • HTML attributes that begin with data- are ignored by the browser and can easily be accessed with JS.
  • localStorage can be used to store information on the client permentantly, and sessionStorage can be used to store information for the duration of the state.
  • navigator.geolocation gets geo data.
  • Worker() can be used to create threads from separate JS files.
  • Service workers can be used to add offline functionality to your app and send push notifications.
  • Websockets are often used for two-way communication, or push messaging.
  • When metadata for a video has been loaded, a loadedmetadata event is fired.
