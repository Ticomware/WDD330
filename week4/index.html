Chapter 8: Forms

  • let form = document.forms['elementId'] is an easy way to get a form from the DOM
  • let element1 = form['elementId'] can be used to access elements within a form
  • element1.value will let you get the text from an input field
  • Use placeholder attribute to use placeholder text
  • Common form controls
    • input (used for fields of many different types: passwords, one-line text fields, check boxes, file uploads, radio buttons, etc.)
    • select (equivalent to WPF combo box)
    • textarea (equivalent to WPF textbox)
    • button (self-explanatory)
  • The autofocus attribute will move focus to an input element when the page loads
  • The maxlength attribute limits the number of characters in an input form
  • A default value for input can be set in JS or HTML (though the form reset() function sets it to the HTML default)
  • An input element has a few different types that it can be set to
    • text
    • password
    • checkbox
      • To link checkboxes together, all checkbox elements must have the same name attribute
      • All checkboxes can be access by using form.nameOfCheckboxes
      • Checkboxes have a checked property to determine if it has been checked or not
      • Checkboxes can be set programmatically in either JS or HTML.
    • radio
      • Like checkboxes, radio buttons need to have the same name attribute to be linked together. You can also access them the same way as checkboxes with dot notation.
      • The value of the radio group is equal to the value of the selected radio button
      • Radio buttons' checked property can be set to true with JS, but will then set all other radio buttons in the group to false. A radio button can be checked by default using HTML.
    • hidden
      • Usually used to pass user settings or non-sensitive information to the backend. It's in plaintext, not encrypted, and easily manipulated by pen-testers, so don't use it for anything too important.
    • file
      • Used to upload files
    • email, search, url, and tel
      • These types all look like a text field, but change the virtual keyboard and give free input validation depending on the type (still validate input on the backend)
    • number
      • This looks like a text field and does change the virtual keyboard, but adds a way to increment values by a step. Use tel for things like zip codes or other numbers that don't make sense to increment.
      • Has attributes min, max, and step
    • color
      • This is a color picker, returns its hex code
    • range
      • This is a slider
    • datetime-local, month, time, week, date
      • These are for picking dates
  • select is used similarly to a combo box, add the multiple attribute to allow the user to select more than one option
    • It has selectedIndex and value to get the value(s) selected
  • textarea works like a text box. Initial value can be set in JS or in the tag in HTML
  • button can have a type of either submit, reset, or button.
  • The required attribute can be applied to any form element, and the page will not submit the request until it is filled out properly.
  • You can validate form input using JS and the event.preventDefault() method
  • Form elements can be disabled using the disabled attribute; consider using this as part of your client-side validation

Chapter 12: Object Oriented Programming in JavaScript

  • Unlike other OOP languages, JS is a prototype-based language. Prototypes are created to base other similar objects on.
  • An example of creating an object using a constructor
    • const object1 = function(){};
      const object2 = new object1();
  • With the ES6 specification, classes can now be created in JS. Using classes is preferred to prototypes as they are strict by default
  • Classes can have static methods, like other OOP languages
  • Methods and properties can be added to classes and objects made from a constructor retroactively by using class.prototype
  • You can check for the prototype of an object using Object.getPrototypeOf(), object.__proto__, or Object.prototype.isPrototypeOf(object)
  • You can set the prototype of an object using Object.setPrototypeOf() (don't know why you'd want to after the object is created, but maybe it's useful in some edge case)
  • hasOwnProperty() checks if the object has its own unique property
  • Prototypes cannot be overwritten if it was declared from a class, but can if it was from a constructor function
  • Use getters and setters for accessing private properties in classes in JS
  • To make a property or method private in JS, use a hashtag as the first character of that method or property (only part of ES2022, probably going to take a while before everyone is using it) or use let _property = value inside a class constructor. Make sure not to write is as this.property or it'll be publicly available.
  • Methods can either be enumerable or non-enumerable; by default, built-in functions are non-enumerable and won't show up as part of a for-in loop. Enumerable functions will show up in a for-in loop and user-defined functions are enumerable by default.
  • You can extend a class from another class by using the extends keyword
  • When extending a class, super(propertyOrMethod) accesses to the parent's properties or methods
  • You can overwrite built-in methods by declaring the method or property with the exact same name in your class
  • Methods and properties can be added to the built-in object prototype, and those values will be part of every object. This is called monkey-patching, but isn't best practice in JS
  • Monkey-patching can be used to add features that are missing in older browsers. You can check if a method or property exists, and if it doesn't overwrite it with a custom implementation
  • Alternatively, you can extend objects into your own custom classes.
  • Instead of using assignment for properties, you can use Object.defineProperty(object, 'propertyName', {writable: false, enumerable: false, value: 'value'});
  • Get and set methods of a property can be overwritten like C# Object.defineProperty(object, 'propertyName', {get() {}, set(value) {}});
  • Object.create(prototype) is another way to create objects from classes
  • Child prototypes can be created as objects like so
    • const child = Object.create(parent);
      child.name = “child”;
  • You can create a function to make it easier to create objects from a class. Remove the method from the object by calling this.methodName = undefined
  • instanceOf() does not work for objects created through classes, only constructor functions
  • Factory functions return an object
  • To chain functions, make sure it returns this
  • You can copy methods from objects by assigning the method to a property

Chapter 15: Modern JavaScript

Modular JavaScript

  • Modules appear to work like small libraries
  • All modules are strict by default, and all modules have their own scope
  • To make a module, create a JS file with a collection of functions and properties. At the end of the file, add: export { method, property, etc }
  • To import properties or methods from a module use import {method, property} from 'module.js'
  • You can import the whole module with import * as name from 'module.js'
  • One module part can be exported by default by using export default
