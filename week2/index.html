Chapter 2: Programming Basics

  • JS has C-style comments
  • C-style syntax
  • Minimize scope of variables whenever possible
  • Try making your variables constants at first, changing them to be mutable only if necessary
  • Avoid starting your variables with an underscore or cash sign, cash sign is used to start variables with popular libs and starting with an underscore conventionally means it's a private variable or method.
  • JS convention is to use camel case naming for variables
  • Backslash escapes characters, like C
  • Typescript helps to overcome the limitations of Javascript being a weak-typed language

Chapter 3: Arrays, Logic, and Loops

  1. Arrays
    • In JS, arrays can have many different types of objects
    • .push(), .pop(), .shift(), and .unshift() are all part of the JS array object
    • Array.splice(index, num of items to remove, [items]) lets you insert an array into another one at index and remove a specified number of items from the array, also at index.
    • The spread operator […] can be used to combine and flatten arrays. Ex: let thing = […foo, …bar];
  2. Sets
    • Sets guarantee that no values are duplicated; they are a collection of unique values.
    • You can convert an array into a set with the following: let thing = new Set([1, 1, 2, 2, 3, 4, 5, 5]);
    • You can convert a set into an array using spread operator or Array.from() method. Ex: thing = […exampleSet] or thing = Array.from(exampleSet)
    • You can strip duplicates from arrays like this: thing = […new Set(oldArray)];
    • A weak set is a set made up of objects that can be easily garbage collected when the object in the set is dereferenced.
  3. Maps
    • List of key-value pairs
    • Maps can be converted into arrays in the same way as sets, with the spread operator or Array.from()
    • A weak map is a map made up of objects that can be easily garbage collected when the object in the map is dereferenced.
  4. Logic
    • Ternary operator syntax: condition ? (code if true) : (code if false)
  5. Loops
    • JS has all the standard loops other programming languages have; for each loops are for(x of y) in JS

Chapter 4: Functions

  • Functions are objects in JS, can be assigned to variables
  • To call a function that was assigned to a variable, invoke it with variableName();
  • Arguments are accessible in a function through explicitly defined variables (i.e. function f(x, y)), through the arguments object, or through the rest operator (ex. function f(…args)). The arguments object is obsolete, and should be replaced with the rest operator.
  • Default parameters works the same as other languages
  • In-line arrow functions are written as follows: (arg1, arg2) => arg1+arg2;
  • Longer arrow functions are written like this: (arg1, arg2) => { return arg1+arg2; }
  • In JS, functions can be called before they are declared in the code file
  • In JS, vars can be referred to before being declared, but will be undefined until assigned to; use let/const instead.
  • If you assign a function to a variable, you can't call the function through the variable until after it is assigned.
  • You can pass a function as an argument in another function
  • Another way to iterate through an array is: Array.forEach( (item,index) => console.log(`${item} is at index ${index}`);
  • Callbacks are used extensively with array manipulation, like other programming languages
