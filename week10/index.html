Week 10: Validating Forms and Using Fetch

Notes

MDN: Validating Forms

  • Always validate input on the server side, AND the client side.
  • Built-in form validation has been extremely important in the development of the modern web; use built-in validation when possible.
  • Built-in form validation comes out of the HTML5 standard, and I largely documented this last week. You can use regular expressions, basic min/max, required, step, etc.
  • textarea doesn't support regex without custom JS
  • JS lets you handle invalid data differently from the built-in functionality. You can check if an input is valid by calling HTMLElement.validity; Has a variety of enums for different reasons why the element is invalid.
  • Use constraint validation API to customize visual validity errors. Using novalidate will prevent the browser's default visual validity from displaying, use your own JS to set your own visual indicators.
  • For custom controls (for example, a calendar) you will need to validate the control yourself.

MDN: Using Fetch

  • Use include to include cookies, credentials with a fetch()
  • CORS is very restrictive, when disabling CORS you are unable to include very much
  • Files can be uploaded using fetch() and FormData.
  • Custom headers can be crafted for fetch() requests by using the Headers object.
  • Fetch promises return a Response object, which contains a Body object which contains the data.
