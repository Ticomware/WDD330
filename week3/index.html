Week 3: Objects, The DOM, and Events

Notes

Object methods, this

  • The this keyword is used to refer to the current object within an object method, similar to this in C#.
  • Anonymous functions' this keyword calls the parents' this.
  • In JS, for backwards compatibility, there is a strict and non-strict mode of the this keyword. A strict this acts the same as this in other languages, but a non-strict this will find the next-closest this that belongs to a parent object.

Chapter 5: Objects

  • Objects in JS work similarly to other OOP languages
  • Object literals in JS can be created without creating a class
  • Bracket notation can be used to access a property (ex. object[result()];)
  • Keys in JS can be programmatically defined like so: const foo = {[result()]: 'value'};
  • Symbols can be used as keys for an object
  • Methods can also be called with bracket notation (ex. object['function']();)
  • You can check if a property or method exists using the in keyword, checking if it does not equal undefined, or using the hasOwnProperty() method.
  • You can iterate through each property and method in a JS object by using a for in loop.
  • object.entries() can be used to get an array of methods and properties.
  • Properties can be removed from an object using the delete keyword.
  • In JS you can wrap a method in an object to namespace it
  • JSON is natively supported in JS and can be parsed into JS objects from a string or turned back into a string with the parse() and stringify() methods respectively.
  • The math and date classes built into JS work similarly to Java.
  • To use RegEx with JS, you need to create a RegExp object with your pattern. No LINQ magic here.
  • To see if a string matches a RegEx pattern, use RegExp.test(string);

Chapter 6: Document Object Model

  • Basic DOM manipulation in JS is discussed here. I am already very familiar with the functions used with the document object to access nodes in a webpage.
  • Some legacy shortcuts I didn't know about:
    • document.body
    • document.images
    • document.links
    • document.anchors
    • document.forms
  • classList lets the programmer add and remove CSS classes from nodes.
  • getElementsByClassName() and getElementsByTagName() return live lists that hurt performance.
  • CSS can be added directly to a node using JS without classes, I struggle to think of how this could be good practice ever.
  • CSS properties are in camel case when access through JS
  • Alternatively, bracket notation can be used to access CSS properties (ex. node.style['font-family'] = 'sans';)
  • getComputedStyle(node) will return a read-only list of the CSS properties that were changed on that node, whether through an inline style or by an external style sheet.

Chapter 7: Events

  • Event Listeners work similarly to C#
  • When a function is called from an event listener, an event object is passed into the function, similar to C#. The event object holds many different properties, including the node that called it and its position on the page in X, Y coordinates.
  • When you use an anonymous function with addEventListener(), it will never be garbage collected. Only use an anonymous function if you intend to use it throughout the entire lifecycle of the page.
  • preventDefault() within a event handler will prevent the node from doing its default action.
  • Events will bubble up the document tree by default, similar to C#
  • Events can be captured instead from the root element down to the individual element by using addEventListener('eventType', eventHandler, true);
  • Events can be stopped from bubbling up by using event.stopPropagation();
  • Events can be assigned to a parent node and the event.target property will be the individual element within the node that was acted upon. Can be good for dynamically created lists.
