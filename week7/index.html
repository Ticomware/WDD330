Chapter 11: Further Functions

  • Methods can have their this property changed to another object by calling the function with the call(newThis, â€¦params) method.
  • Custom properties can be added to methods like any other object in JS.
  • As a consequence of the above two features, we can add a cache property to a method to cache the results of a calculation.
  • JS has a feature called Immediately Invoked Function Expressions (IIFEs) where a function can be defined and immediately executed. This can be used to create a function without defining new global variables.
  • IIFEs can be used to program temporary variables, since variables cannot be dereferenced without disposing of the entire scope.
  • With ES6, you can make code blocks. Variables in a code block are in their own scope, and are destroyed when the scope is destroyed.
  • 'use strict' can be used in IIFEs.
  • Functions can be redefined within themselves, usually used to change a functions behavior after it is run for the first time. This is needed often as part of a function that utilizes feature checking.
  • Recursive functions are available
  • JS is single-threaded, but callbacks are asynchronous
  • Async functions can be wrapped in a Promise object, were the function either resolves or rejects. When a Promise is rejected or resolves, it returns a value.
  • Calling the then() method on a promise object can control code flow depending on if it was resolved or rejected. The catch() method sets the code to run in case of a rejection only.
  • Async/await exists in JS and works similarly to C#.
  • Use callbacks to reduce the number of unique functions in your program.
  • Functions can return functions.
  • Variables have block scope in JS, as we've already talked about.
  • When a function is returned from another function and stored in a variable, the parent function and all children functions are stored in that variable, along with any variables used within the parent or child functions.
  • In JS, iterators are called generators. They are defined by adding an asterisk immediately after function keyword (no spaces.) Yield results like C#.
  • JS doesn't have a strict coding style, and can be used with OO, functional, and procedural programming styles.
  • Returning a function based off of an input can simplify your code when using a functional programming approach
  • Currying is when a function is called without all required arguments, so the function returns a function; saving the arguments passed into it and requiring the missing arguments to run.
  • A function can be written to automatically curry a function, saving you from hard-coding currying instructions on every function.

Chapter 13: AJAX

  • The fetch() method is a global promise object used to get a response from a URL.
  • The Response object returned from fetch() has an ok property where if true, it got an HTTP code of 200-299. Needed for checking for 404s and other status codes.
  • Response also has the properties of headers, url, redirected, and type.
  • Response objects can format the data they received with the following methods
    • response.text() gets the data returned when fetching the url as a string.
    • response.blob() gets binary data from the URL, returns after the data is downloaded.
    • response.json() gets data and converts it into a JS object, assuming it's JSON data.
  • Response objects can be created using a constructor function, can be used for testing or building APIs.
  • Request objects can give you more control over how to fetch data from a URL, and has the following properties
    • url
    • method
    • headers
    • mode
    • cache
    • credentials
    • redirect
    • body
  • Headers objects are used to make HTTP headers as part of a fetch request.
  • Data can also be sent using the fetch api using a request object, the data to send will be the body property.
  • A FormData object creates an object automatically from all the form data in a form.
  • FormData objects handle file uploads very well.
  • jQuery is a common JS library.
