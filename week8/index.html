Chapter 8: Transforms and Transitions

  • transform: translate(x, y) – Moves an element relative to its current position
  • OR you can use translateX() and translateY()
  • transform: scale(x,y) – Scales an element
  • OR you can use scaleX() and scaleY()
  • Multiple transform declarations override each other, chain your transformation properties with a space-separated list
  • transform has no impact on document layout
  • transform: rotate(deg) – Rotates an element
  • transform: skew(x, y) – Skews an element
  • transform-origin: x y – Changes the point where transformations occur on object
  • transition has several properties, that I'm already familiar with
  • visibility: visible/hidden can be transitioned, but not display:none
  • Must specify transition-duration for transitions to display
  • transition-timing-function lets you ease transitions
  • transition-delay lets you set a delay
  • Negative delays can be used to start a transition part-way through
  • CSS animations have support for keyframes, has some weird syntax. Read more here
  • CSS animations can be reversed easily

Chapter 12: Canvas, SVG, and Drag and Drop

  • In HTML5, the width and height of canvas must be set in HTML. CSS height/width will be ignored.
  • Canvas is a powerful tool that allows JS to draw to it. You need to grab a context object from canvas to draw to it.
  • Drawing an arc in a canvas requires you to use radians, because Apple
  • Canvas can be used to programmatically alter images and video
  • Changing the height or width of a canvas will clear its contents, because Apple
  • SVG is supported in HTML5
  • They use a 3rd party library to animate an SVG. Not interested.
  • Canvas is faster, SVG is more accessible
  • HTML5 supports drag and drop, syntax is too complicated to explain here. Basically, you link drag events together and get data from the object from the arguments.
